# liteparse

## 项目定位
liteparse 是 run-llama 开源的文档解析工具，强调“快速、可用、开放”的文本抽取能力，面向 RAG 与 Agent 数据预处理场景。

- GitHub: https://github.com/run-llama/liteparse

## 用法

### 1. 安装与运行
- 按仓库说明安装二进制或源码编译。
- 输入文档后输出结构化文本结果用于后续索引。

### 2. 接入检索链路
- 解析后结果可直接进入向量库或关键词检索系统。

## 原理
- 通过高性能解析实现对文档内容的快速抽取。
- 强调“解析速度 + 可读输出”的平衡，服务检索前处理。
- 与 LlamaIndex 等生态结合，作为 ingestion 层组件。

## 价值
- 在大规模文档入库前阶段显著缩短处理时间。
- 对需要快速迭代知识库的团队有实际收益。
- 适合作为多解析器策略中的高吞吐选项。

## 风险边界
- 某些复杂 PDF/版式仍可能出现结构识别误差。
- 高速解析在边缘场景下可能牺牲部分精细还原度。
- 单解析器依赖会放大异常样本的影响。

## 补充建议
- 与第二解析器做抽样比对，建立质量告警阈值。
- 对关键字段（标题、表格、引用）做专项回归检查。
- 在 RAG 评估中加入“解析误差对召回率影响”指标。
