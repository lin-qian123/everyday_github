# 12-factor-agents

## 定位
`humanlayer/12-factor-agents` 提出可生产化的 Agent 软件工程原则，类似把传统“12-Factor App”思想迁移到 LLM 应用。

## 用法
1. 先通读原则清单，识别与你当前系统冲突的点。
2. 从可落地项优先改造：配置分离、可观测性、失败恢复。
3. 将原则映射到 CI/CD 与运行手册，形成团队执行规范。

## 原理
- 以“可部署、可观测、可恢复”为核心约束 Agent 系统。
- 将提示词、工具调用、状态管理纳入工程化边界。
- 强调在不确定模型行为下仍可维护的系统设计。

## 价值
- 把“能跑 demo”提升为“可持续运营”。
- 降低线上不可解释故障的排障成本。
- 对企业级合规与变更管理更友好。

## 风险边界
- 原则是框架，不是即插即用模板。
- 过度追求“完整遵循”会牺牲交付速度。
- 不同行业监管要求差异大，需二次定制。

## 补充建议
- 按“风险最高环节”分阶段引入，不要一次性重构。
- 给每条原则配可验证指标（SLO、错误率、恢复时长）。
- 和安全评审、日志审计一起推进。
