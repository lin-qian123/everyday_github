# PageIndex

## 项目定位
PageIndex 是一套以“推理优先”的 RAG 检索方案，主张用分层树与推理搜索替代传统向量库与 top-k 检索。它把“检索”变成“多步推理 + 结构化搜索”，强调可解释、可控与更高的召回与精准性。

来源：
- https://github.com/VectifyAI/pageindex-mcp
- https://github.com/vectifyai
- https://sejiwork.com/en/blog/post/github-trending-daily-2026-02-24

## 核心理念
- 向量检索不是必需：PageIndex 走“无向量、无切片、无 top-k”的路线，用树结构与推理搜索直接定位内容。
- 检索即推理：把“找资料”当作“逐步决策”，每一步都基于上下文做更细粒度的选择。
- 结构优先：先构建层级索引（章节/主题树），再在树上做多步探索。

## 原理概述（基于官方说明）
1. 构建层级树（Tree Index）
   - 将知识库结构化为层级树，每个节点代表主题或子主题。
2. 推理搜索（Reasoned Search）
   - 模型基于问题逐步选择路径，从粗到细定位关键信息。
3. 输出与可解释性
   - 由于检索路径是显式的，能提供更清晰的“为什么找到这些内容”。

## 适用场景
- 结构化文档（手册、报告、知识库、技术文档）。
- 需要可解释检索路径的业务（合规、审计、客户支持）。
- 对“向量召回噪声”不满意，希望更可控的检索。

## 快速体验（MCP 方式）
PageIndex 提供 MCP 服务器，适合在 Claude Desktop 等支持 MCP 的客户端中直接使用。

1. 下载 MCP 包
   - https://pageindex.ai/mcp （官方页面）
2. 在 Claude Desktop 中添加 MCP Server
   - 选择本地 `.mcpb` 包并完成安装
3. 使用（示例）
   - 在对话里通过 MCP 工具完成索引与检索（以具体客户端配置为准）

## 使用建议
- 文档结构越清晰，树索引的效果越稳定。
- 对“解释性”和“稳定性”要求高的场景优先考虑。
- 如果数据高度非结构化或缺乏层级关系，可能需要先做清洗与组织。
