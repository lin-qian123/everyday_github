# fence

## 定位

`fence` 是 Claude Code skill，围绕 Chesterton's fence 原则调查奇怪代码为何存在、能否安全删除。

截至 2026-07-23，GitHub API 显示其创建于 2026-07-22，约 1 star；属早期开发者信号。

## 用法

将可疑代码位置交给流程，由其检查 blame、旧提交、PR 和历史讨论，再输出删除风险与证据。

## 原理

把 Git 历史和项目考古变成删除前的固定步骤，避免 agent 仅因局部代码“不优雅”就重构。

## 价值

- 对遗留系统和安全相关分支特别有用。
- 有助于让删除决策可解释、可复核。

## 风险边界

- 历史记录可能缺失、错误或含敏感信息。
- 理解历史不替代运行测试和产品确认。

## 补充建议

把结论与最小删除 PR、回归测试和回滚方案绑定。
