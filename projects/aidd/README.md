# aidd

## 定位

`aidd` 是本地优先的 AI Development Director，试图作为跨 CLI coding agent 的监督与任务控制台。

截至 2026-07-23，GitHub API 显示其创建于 2026-07-22，约 1 star；属早期开发者信号。

## 用法

按项目文档配置可用 CLI 后端、工作区和任务入口，先在单仓库、低权限模式下验证调度行为。

## 原理

将多个 agent 的任务、状态和监督动作放进一个本地控制面，而把模型执行保留在各自 CLI 后端。

## 价值

- 代表多宿主 coding agent 的控制面需求。
- 本地优先对需要保留代码与日志边界的团队较有吸引力。

## 风险边界

- 调度器往往需要读取工作区、进程状态甚至凭据，权限面较大。
- 早期项目的兼容性、并发安全和恢复策略需实测。

## 补充建议

优先验证 worktree 隔离、退出恢复和审计日志，不要直接给予生产仓库写权限。
