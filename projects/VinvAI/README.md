# VinvAI

## 定位

`VinvAI` 是面向 Python 后端的本地 coding-agent 验证层：它记录真实运行轨迹、建立代码图谱，并经由 MCP 把证据交给现有 agent；修复后再用独立启动与验收检查验证“完成”声明。截至 2026-07-24，仓库创建于 2026-07-23，约 20 stars，仍属早期项目。

## 用法

克隆仓库后执行 `./install.sh`，再打开待分析的 Python 项目；先在隔离的开发环境中让其发现服务、建立索引并记录一次可重复请求。将其 MCP 工具接入已有 coding agent 后，要求 agent 依据 trace 与符号证据定位问题，最后检查重放、端口探测和验收测试的结果。

## 原理

项目以零代码修改的 Python tracing 收集调用、参数摘要、返回值与错误；再用本地嵌入、语义检索和调用图将运行 trace 关联到具体符号。它将修复前生成的验收检查与修复后的独立执行分开，避免 agent 自评自证。

## 价值

- 把 coding agent 的自然语言“已完成”转成可审计的运行证据。
- 本地运行、复用既有 CLI agent，适合不希望另配模型 API 的团队。
- 为 trace、源码、日志与指标之间的联合检索提供了具体样本。

## 风险边界

- 当前运行时证据以 Python 后端为主，TypeScript/Go 等栈的能力并不等价。
- trace 可能含请求参数和敏感业务数据，必须先审查脱敏与保留策略。
- 安装会构建多种组件并下载本地嵌入模型；项目为 v0.0.x，资源占用和恢复行为应先实测。

## 补充建议

先用无敏感数据的测试服务验证 trace 范围、MCP 暴露面和回滚，再考虑接入真实仓库。验收测试应由团队维护，而不应只依赖工具自动生成。
