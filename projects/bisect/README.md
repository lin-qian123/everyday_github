# bisect

## 定位

`bisect` 是 Claude Code skill，将自然语言 bug 描述转为检查命令，并在隔离 worktree 中驱动 `git bisect` 找到引入问题的提交。

截至 2026-07-23，GitHub API 显示其创建于 2026-07-22，约 1 star；属早期开发者信号。

## 用法

提供可稳定判定的故障条件和已知好/坏版本；在隔离工作树中执行二分，最后人工核验归因。

## 原理

先构造机器可判定的测试，再利用 Git 二分将回归定位从线性翻历史变为对数级搜索。

## 价值

- 适合把 agent 的历史检索能力转化为可重跑的缺陷定位。
- worktree 隔离有助于避免破坏当前工作区。

## 风险边界

- 前提是测试结果稳定；非确定性 bug 可能误导二分。
- 自动 checkout/执行历史代码应在隔离环境、低权限下完成。

## 补充建议

先用 `repro` 固定最小失败，再运行二分；将最终修复附带回归测试。
