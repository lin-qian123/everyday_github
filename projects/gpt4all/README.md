# gpt4all

## 定位
`nomic-ai/gpt4all` 是一个强调本地可运行、离线可用的 LLM 运行与分发项目，适合对隐私和离线能力有要求的个人与团队。

## 用法
1. 访问项目主页并下载桌面应用或按文档安装 CLI/SDK。
2. 选择本地模型（如通用对话、小参数指令模型）并在本机加载。
3. 按需启用本地文档问答（LocalDocs）做私有知识检索。
4. 在业务中优先用于“可离线、低敏感”场景，避免直接承载高风险决策。

## 原理
- 通过本地推理栈在用户设备执行模型推理，减少云端依赖。
- 结合向量检索能力将本地文档切片后召回，再交给本地模型生成回答。
- 核心价值在于“部署门槛低 + 数据留在本地”。

## 价值
- 隐私友好：敏感数据可不出设备。
- 成本可控：降低 API 调用成本与网络依赖。
- 教学/原型友好：适合做本地 AI 助手与 RAG 演示。

## 风险边界
- 本地模型能力受参数规模和硬件限制，复杂推理质量不稳定。
- 检索质量高度依赖文档清洗和切片策略。
- 离线部署不等于绝对安全，仍需终端安全和访问控制。

## 补充建议
- 在生产前做小规模基准评估（准确率、延迟、资源占用）。
- 建立“本地可答/云端升级”双通道，按任务复杂度路由。
- 对关键输出增加规则校验与人工复核。
