# content-core

## 定位

`content-core` 是生产型 AI 平台内核，组合 LLM 抽象、workflow、RAG、MCP、FastAPI、评测与闭环反馈。

截至 2026-07-23，GitHub API 显示其创建于 2026-07-21，约 3 stars；属早期开发者信号。

## 用法

按仓库模块选择接入 LLM provider、知识检索、工作流和评测组件；应先在隔离环境核验配置与数据模型。

## 原理

把 AI 应用常见横切能力收敛为平台层，业务应用通过统一接口编排模型、检索、工具与反馈。

## 价值

- 适合观察“应用内核化”的架构取舍。
- 将评测与反馈放进主链路，而非上线后的旁路。

## 风险边界

- 大而全内核可能带来耦合、升级和供应商适配成本。
- RAG/MCP 配置可能涉及用户数据和外部凭据。

## 补充建议

先验证最小垂直切片的检索质量、观测性和权限隔离，再决定是否采用平台化抽象。
