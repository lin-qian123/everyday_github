# pi

## 项目定位
pi 是一个 AI Agent 工具链方向项目，强调命令行入口、统一模型接口与多端交互（含 TUI / Web / Bot）能力，目标是降低搭建代理系统的工程门槛。

- GitHub: https://github.com/earendil-works/pi
- Trending 参考: https://github.com/trending

## 用法
1. 克隆仓库并按官方步骤安装。
2. 配置模型 API 与基础工具。
3. 通过 CLI/TUI 启动 Agent，会话中接入所需插件或工作流。
4. 根据任务类型选择本地执行或远程服务模式。

## 原理
- 统一接口层：屏蔽不同模型供应商的接入差异。
- 会话控制层：管理 Agent 上下文、工具调用与输出归档。
- 运行时扩展层：支持 bot/web 端接入，拓展交互场景。

## 价值
- 从“模型调用”提升到“可运营的代理系统”。
- 适合个人开发者和小团队做快速原型验证。

## 风险边界
- 通用代理框架不等于业务可用系统，仍需大量场景适配。
- 工具调用链越长，稳定性和可观测性压力越大。
- 多模型混用时需注意数据安全与成本失控。

## 补充建议
- 先定义最小可用工作流，再增量扩展插件。
- 为关键任务建立可回放日志和失败复盘机制。
- 对模型成本设预算上限与告警阈值。
