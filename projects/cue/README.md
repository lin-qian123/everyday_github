# cue

## 定位

`cue` 是一个开源 macOS AI copilot，主打悬浮在屏幕上、读取屏幕和会议音频、在屏幕共享中保持隐藏，并由用户自带模型密钥驱动。它明显对标 Cluely 一类“会议/面试辅助”工具，但选择开源与 BYOK 路线。

截至 2026-07-18 抓取时，仓库约 `479` stars、`98` forks，主要语言为 JavaScript，许可证为 `GPL-3.0`。

## 用法

- 普通用户可下载应用包安装。
- 开发者可从源码运行：

```bash
npm install
npm start
```

- 打包 macOS 应用：

```bash
npm run pack
```

README 要求 Node.js 18+，并提供首次启动的 1 分钟设置流程。

## 原理

项目以桌面悬浮层为交互入口，结合屏幕内容、会议/音频输入和用户配置的 LLM provider，把实时场景转为可回答、可提示、可辅助决策的上下文。它更像“桌面级 ambient copilot”，而不是传统聊天客户端。

## 价值

- 代表 AI 助手从聊天窗口进入真实桌面场景的趋势。
- BYOK 与开源实现降低了黑盒 SaaS 的不透明性。
- 对会议记录、面试准备、屏幕问答和实时辅助场景有参考价值。

## 风险边界

- “隐藏于屏幕共享”能力存在明显伦理和合规风险，尤其在考试、面试、受控会议中。
- 读取屏幕和音频意味着极高隐私敏感度，必须明确数据流、模型端点和日志策略。
- GPL-3.0 对再分发和集成有约束。

## 补充建议

- 后续观察其是否补充权限提示、场景限制和企业合规模式。
- 不建议把它作为“绕过会议规则”的工具宣传，更适合研究桌面 AI copilot 的交互模式和隐私边界。
