# financial-services

概览
`anthropics/financial-services` 是一个围绕金融业务场景构建的 AI 工作流与插件示例集合。它展示了如何把 LLM 能力嵌入到合规、分析、运营等高约束行业流程中，而不是只做通用聊天。

为什么值得关注
- 行业化模板：直接面向金融服务流程，迁移到企业场景更自然。
- 工具化接口：通过插件/MCP 等方式将模型接入真实系统。
- 可治理导向：强调流程边界、权限控制、审计与可追踪性。

如何使用
1. 克隆仓库后先阅读 `managed-agent-cookbooks` 与 `plugins` 目录说明。
2. 选择一个业务流程样例（如客服、分析、流程编排）进行本地演练。
3. 把企业内部数据源或 API 以工具接口形式接入。
4. 增加日志、权限和审计策略后，再推进到真实业务链路。

原理拆解
- 任务编排：将复杂金融流程拆成多步可验证子任务。
- 工具调用：模型通过受控工具访问外部系统，而非直接“自由发挥”。
- 风险控制：在关键节点加入校验、回退与人工确认。
- 结构化输出：优先可解析结果，便于后续系统消费。

适用人群
- 需要做金融 AI 落地 PoC 的工程团队。
- 关注 MCP/插件化工作流的产品与架构人员。
- 对“合规 + 自动化”平衡有强需求的企业团队。

补充观察
- 金融行业的 AI 竞争核心不是单次回答质量，而是“流程正确率 + 合规可解释性 + 交付速度”。
- 这类仓库可作为“行业 Agent 工程模板库”来复用，而不只是 demo。
