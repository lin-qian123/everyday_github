# agentmemory

## 定位
`rohitg00/agentmemory` 是面向 AI 编码代理的持久化记忆层，目标是在多轮会话和跨项目任务中减少上下文丢失。

## 用法
1. 打开仓库并按官方安装方式接入（通常作为本地服务或库被代理调用）。
2. 在代理运行链路中启用“写入记忆/检索记忆”步骤。
3. 用真实任务验证：同一问题跨轮追问时是否能复用历史决策与偏好。

## 原理
- 将“短期上下文”外置为可检索存储。
- 通过结构化索引（如任务、项目、偏好、约束）提升召回质量。
- 在提示词编排层决定何时读写，控制噪声注入。

## 价值
- 降低重复解释成本。
- 提升长流程任务一致性。
- 对多代理协作场景更友好。

## 风险边界
- 记忆污染会放大错误结论。
- 若缺少过期策略，旧知识会误导当前任务。
- 涉及隐私数据时需额外合规与脱敏。

## 补充建议
- 先定义记忆 schema（事实、偏好、决策、证据）。
- 增加 TTL/置信度字段，避免“永久真理化”。
- 对关键结论保留来源链接，支持回溯。
