# n8n-mcp

- GitHub: <https://github.com/czlonkowski/n8n-mcp>
- 定位：把 n8n 节点知识库以 MCP Server 形式暴露给 AI 助手，用于自动设计/检查工作流。

## 它解决什么问题

很多 AI 助手在生成 n8n 工作流时，常见问题是“节点参数不全、字段名错误、版本不匹配”。`n8n-mcp` 通过结构化节点元数据和模板库，降低这类错误率。

## 用法（推荐）

1. 按仓库说明启动 MCP Server。
2. 在 Claude/Codex/Cursor 等支持 MCP 的环境中注册该 server。
3. 先让助手做“模板优先”检索，再做工作流生成。
4. 对生成结果执行多级校验（最小校验 -> 全量校验 -> 端到端校验）。

## 工作原理

- 数据层：聚合 n8n 节点文档、参数 schema、操作清单、模板示例。
- 协议层：通过 MCP 统一给上层助手调用（查询节点、查模板、校验工作流）。
- 生成层：LLM 在调用 MCP 工具后生成更可执行的 n8n JSON/workflow 结构。

## 价值与注意事项

- 价值：
  - 提高自动生成工作流的可执行率；
  - 降低“看起来对、运行时报错”的概率；
  - 适合自动化团队做标准化交付。
- 注意事项：
  - 节点生态更新快，需关注版本同步；
  - 生产环境必须补权限控制和审计日志；
  - 默认参数不可盲信，关键字段应显式声明。
