## 1. 项目定位
`sim` 是一个开源的 AI agent 编排平台，定位为“AI workforce 的中控层”。
它把模型、工具、数据源和外部集成连接在一个可视化画布上，强调从单 agent 到多 agent 工作流的落地。

## 2. 基础用法
1. 按仓库文档安装并启动本地环境（monorepo + 多服务组件）。
2. 在可视化画布创建 workflow：
   - 连接模型节点
   - 连接工具/API 节点
   - 加入条件判断和错误处理节点
3. 运行并观察每一步输出，基于日志调优提示词与节点参数。
4. 将稳定流程部署到团队环境，结合权限、配额和监控策略运营。

## 3. 原理理解
- 编排层：DAG/流程图式任务分解，把复杂任务拆成可执行步骤。
- 执行层：每个节点可映射到模型调用、工具调用或数据处理动作。
- 协同层：支持多 agent 在同一流程中分工，形成“计划-执行-校验”闭环。
- 集成层：通过丰富连接器接入业务系统，避免 agent 成为“孤岛能力”。

这类平台的关键价值不是“更聪明的回答”，而是“更稳定的执行链路”。

## 4. 典型场景
- 自动化客服与工单分派。
- 销售线索筛选与外呼前研究。
- 内容生产流水线（选题、草拟、审校、发布）。
- 企业内部知识检索 + 流程自动执行。

## 5. 风险与注意点
- 低门槛编排会放大误配置风险，必须建立发布前测试门禁。
- 多连接器环境下，密钥管理与最小权限控制是基础项。
- 工作流越长，越要重视失败重试、幂等性和可观测性。
