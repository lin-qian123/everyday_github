# ai-engineering-from-scratch

## 定位
`rohitg00/ai-engineering-from-scratch` 是一个面向工程落地的 AI 系统化课程仓库，强调“从数学与底层实现到 Agent/生产部署”的端到端训练路径。

## 用法
1. 克隆仓库后先读根目录 `README.md` 与 `ROADMAP.md`，明确 20 个阶段（phases）的学习顺序。
2. 按阶段执行：先做理论/推导，再完成代码与测试，再沉淀可复用产物（prompt、skill、agent、脚本）。
3. 若是团队学习，可按阶段拆分负责人，用 `outputs/` 与 `projects/` 做阶段性交付验收。

## 原理
- 采用“分阶段递进”结构：从基础数学、机器学习、深度学习，逐步到 LLM、工具调用、多代理和生产化。
- 每节课强调“可运行产物”而非纯概念阅读，降低知识-工程落差。
- 通过统一学习闭环（问题定义 -> 推导/实现 -> 验证 -> 复盘）强化可迁移能力。

## 价值
- 适合希望系统补齐 AI 工程能力的开发者与技术团队。
- 对“会调 API 但缺底层理解”的人群有明显提升价值。
- 课程化结构便于企业内训和长期学习追踪。

## 风险边界
- 学习路径较长，时间成本高，不适合只追求短期 demo 的场景。
- 仓库快速增长时，阶段质量和更新节奏可能不均衡。
- 热度高不代表每个模块都已稳定完备，需结合 issue/commit 活跃度判断。

## 补充建议
- 建议先选 2-3 个与你当前业务最相关的阶段做“试跑”，再决定是否全程投入。
- 结合本地项目做镜像实践（例如把某一阶段方法迁移到真实代码库）会显著提高收益。
- 对关键模块建立“验收清单”：可复现、可解释、可扩展。
