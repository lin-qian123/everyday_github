# lapian-notes

<!-- markdownlint-disable MD013 -->

## 定位

`lapian-notes` 是一个面向影视创作者和叙事研究者的本地拉片工具。它把一部电影变成可编辑的 AI 辅助分析笔记：本地抽帧、字幕处理、AI 分析包、剧情泳道时间轴、结构树、情绪曲线和段落深拆。

截至 2026-07-13，GitHub API 显示该仓库约 281 stars、35 forks，创建于 2026-07-08，许可证为 MIT。它不是通用视频生成工具，而是偏“影视文本细读 + 本地资料整理 + AI 分析回填”的专业工作流。

## 用法

非开发者可下载 ZIP 后直接运行：

```text
Windows: run.bat
macOS: run.command
```

开发者可本地启动：

```bash
npm install
npm run dev
```

典型流程是导入影片，工具本地抽帧并读取字幕，生成可发给 ChatGPT 等模型的分析包；用户把 AI 返回的 JSON 导回工具后，再继续编辑段落、时间轴、结构树和导出结果。

## 原理

项目把视频理解拆成两段：确定性本地处理负责抽帧、字幕、播放联动和笔记存储；开放模型负责结构分析、剧情线命名、段落拆解和情绪曲线。这样既避免绑定单一 AI API，也让影片与笔记主要留在本机。

## 价值

- 把“看电影写笔记”变成可反复编辑的结构化工作流。
- 对中文影视创作者、影评、编剧学习和游戏叙事分析更直接。
- 不强制上传影片到云端，降低素材隐私和版权风险。
- 和 `watch-skill`、`claude-video`、`FableCut` 不同，它更偏研究笔记，而不是 agent 工具调用或视频剪辑。

## 风险边界

- AI 分析结果依赖用户选择的模型，剧情结构和情绪曲线仍需人工校验。
- 影片版权、字幕来源和二次传播必须由使用者自行负责。
- 浏览器本地存储和 IndexedDB 适合个人项目，不等于团队级素材资产管理。
- UI 当前偏中文用户，对国际化和多人协作的支持有限。

## 补充建议

- 适合用于电影、广告、游戏 cutscene、短片和课程案例的深度拆解。
- 若要做团队工作流，需要补充项目文件规范、版本管理和导出格式约定。
- 可持续观察它是否会接入更多本地视频理解模型，减少外部模型往返。
