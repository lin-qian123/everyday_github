# sigbound

## 定位

`sigbound` 用普通 Git 在同一仓库并行运行多个 AI coding agent，并尝试只自动合并构建和测试均通过的改动。

截至 2026-07-23，GitHub API 显示其创建于 2026-07-21，约 2 stars；属早期开发者信号。

## 用法

配置可用模型后，将任务分给并行 agent；每个改动应进入独立分支或 worktree，并以明确的构建/测试门槛决定合并。

## 原理

把并行探索与合并策略分开：Git 承担隔离和历史，CI/测试作为最小准入信号。

## 价值

- 直接回应多 agent 并行开发的冲突和合并瓶颈。
- BYOM 设计减少对单一模型供应商的绑定。

## 风险边界

- 通过测试不保证需求正确、安全或可维护。
- 自动合并需防范共享文件冲突、隐藏副作用和凭据泄露。

## 补充建议

先在低风险仓库试用，并将 lint、测试、审查和回滚作为独立门槛。
