# minos

## 定位

`minos` 是一组为 coding agent 编写的硬规则，主张用可持续执行的约束替代含糊的“最佳实践”。

截至 2026-07-23，GitHub API 显示其创建于 2026-07-21，约 5 stars；属早期开发者信号。

## 用法

阅读规则后选择性放入项目指令文件，并结合现有测试、CI 和权限策略执行，不建议整包盲目覆盖。

## 原理

通过短而明确的不可违反规则限制 agent 的修改、验证和交付行为，减少长提示中的规则漂移。

## 价值

- 适合把团队已经验证过的工程底线固化到 agent workflow。
- 与文件化 harness、代码审查和验收清单互补。

## 风险边界

- 强规则可能与不同仓库的发布流程冲突。
- 规则文本本身不提供运行时隔离或测试保证。

## 补充建议

优先移植与本仓库风险直接相关的几条，并为每条规则提供可执行验证。
