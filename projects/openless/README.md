# openless

## 项目定位
`openless` 是一个开源、跨平台的语音输入工具，核心卖点不是普通听写，而是把自然口语实时整理成更可直接使用的文本，尤其适合生成结构化提示词、长消息、邮件和开发协作文案。

- GitHub：https://github.com/Open-Less/openless
- 官网：https://openless.top/
- 适用人群：重度使用 ChatGPT、Claude、Cursor、Notion、邮件和即时通信工具，希望通过语音更快生成高质量文本或提示词的个人用户与内容/开发工作者。

## 用法

官方 README 给出的使用模式非常直接：

1. 安装桌面应用。
2. 在任意文本输入框里放置光标。
3. 按住全局热键开始说话。
4. 松开后由 ASR + LLM 自动完成转写、润色和插入。

公开文档里强调的典型场景包括：

- 口述 prompt，再直接粘贴到 ChatGPT / Claude / Cursor。
- 口述邮件、PR 描述、规格说明、长消息。
- 选择不同的 style pack，把输出约束成更适合当前任务的文本风格。

它不是语音助手，不会自己回答问题；它只负责把“说出来的话”变成更可用的文字。

## 原理

`openless` 的核心流水线很清楚：

- 录音层：通过全局热键启动/停止录音。
- 转写层：接云端或本地 ASR，把语音变成初始文本。
- 润色层：把初始文本交给 LLM 做清理、结构化和风格转换。
- 插入层：把结果流式写回当前光标位置，失败时退回剪贴板。

它和传统 dictation 工具的最大差异，在于“结构化 prompt 模式”与“style pack 市场”：

- 结构化 prompt：把随口说出的需求整理成上下文更完整的提示词。
- Style pack：通过不同 system prompt 固化不同写作风格。
- Streaming insertion：不是等整段生成完再粘贴，而是边处理边写入，降低体感延迟。

这说明它本质上是“语音输入 + 提示词工程 + 桌面插入”的组合工具。

## 价值

- 对 AI 重度用户，最直接的价值是把“说话生成 prompt”做成可日用的低摩擦输入方式。
- 对开发和内容工作者，它比普通转写更进一步，因为输出目标是“能直接发/能直接贴/能直接提交”的文本。
- 对注重隐私和可控性的用户，开源、自带本地/自带密钥路线，比纯 SaaS 语音工具更可审查。

## 风险边界

- 它提升的是输入效率，不会替你解决事实正确性、业务判断或最终文案质量问题。
- 语音输入链路依赖热键、麦克风权限、ASR 稳定性和目标应用插入兼容性，桌面细节问题会比较多。
- style pack 和提示词润色越强，越要留意是否把原意过度改写。

## 补充建议

- 真正值得先验证的是“对你自己的高频输入场景是否明显提速”，例如 prompt、PR 描述、邮件，而不是泛泛试一遍。
- 如果团队要采用这类工具，优先评估本地数据留存、模型密钥管理、日志与历史记录边界。
- 从趋势看，`openless` 说明 AI 工具竞争正从“模型回答”外溢到“输入层提效”，语音到结构化文本可能会成为一条被低估的工作流赛道。
