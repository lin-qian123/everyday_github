# LongLive

## 项目定位
LongLive（LongLive 2.0）聚焦长视频生成基础设施，目标是提升多镜头、长时段视频生成的可控性与一致性，属于生成式视频基础设施赛道。

- GitHub: https://github.com/NVlabs/LongLive
- Trending 参考: https://github.com/trending

## 用法
1. 依据仓库说明准备环境与模型权重。
2. 配置视频长度、镜头策略与生成参数。
3. 执行推理任务并检查时序连贯性和视觉漂移。
4. 在后处理阶段进行剪辑与质量筛选。

## 原理
- 长时序建模：强调跨镜头语义保持和运动一致性。
- 生成基础设施：围绕训练/推理管线优化吞吐与稳定性。
- 质量控制：通过策略约束减少长视频中的崩坏与漂移。

## 价值
- 对故事化视频、教学视频、演示视频生产有工程意义。
- 为后续多模态 Agent 视频能力提供底层积木。

## 风险边界
- 长视频生成成本高、算力要求高。
- 内容真实性与版权边界需额外审查。
- 生成稳定性仍依赖模型版本与硬件条件。

## 补充建议
- 先做短片段验证，再逐步扩展到长时长。
- 建立质量打分指标（连贯性、细节稳定性、指令遵循）。
- 结合人工审核，避免直接自动发布。
