# RAG-OS

## 定位

`RAG-OS` 是自托管的个人 agent/RAG 系统，主张用 Git + Markdown 知识库、常驻任务调度和本地硬件构建个人 AI 工作台。

截至 2026-07-23，GitHub API 显示其创建于 2026-07-21，约 2 stars；属早期开发者信号。

## 用法

部署到自有硬件，连接本地知识库和 coding-task dispatcher；先限定数据目录、网络和任务权限。

## 原理

将可读的 Markdown/Git 作为长期记忆和审计层，agent 在此基础上检索上下文、分发任务和维护状态。

## 价值

- 对个人 AI 基础设施和可迁移记忆提供一条本地优先路线。
- Git 历史有利于复盘知识和指令如何演化。

## 风险边界

- 常驻 agent 需要严格的操作授权与资源配额。
- Git 文档并不天然解决检索质量、私密数据和冲突合并问题。

## 补充建议

先以只读检索模式试运行，再逐步开放任务执行和写入权限。
