# agent-skills

概览
`agent-skills` 聚焦 AI 编码代理在真实工程中的“技能化”能力建设，即把最佳实践沉淀成可复用、可组合、可审计的技能单元，减少代理行为漂移。

为什么重要
- 把“提示词技巧”升级为“工程化技能资产”。
- 让代理在不同仓库里保持更稳定的编码行为。
- 便于团队共享流程规范，而不依赖个人经验。

如何使用
1. 选择与你的研发流程匹配的技能集合。
2. 将技能文档接入日常代理提示上下文。
3. 按项目阶段启用不同技能（如调试、评审、发布）。
4. 持续回收失败案例，迭代技能条目。

原理拆解
- 显式规则：把隐性经验转成明确动作与边界。
- 阶段化调用：在不同任务阶段加载不同能力模块。
- 反馈闭环：通过代码评审和故障复盘修正技能。
- 组织沉淀：把个人方法论升级为团队共享资产。

适用人群
- 多人协作且大量使用 AI 编码助手的团队。
- 希望降低“同任务不同人输出差异”的项目。
- 需要可追踪代理行为标准的工程管理者。

补充观察
- 技能库是 2026 年 AI 工程工具的关键分层，地位接近过去的软件框架与脚手架。
- 真正有效的技能应当包含“触发条件、失败模式、回退策略”，而不是泛泛建议。
