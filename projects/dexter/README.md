# Dexter — 自动化“深度研究”AI 代理

## 项目简介
Dexter 是一个面向“深度研究”的多智能体系统，目标是把复杂问题拆解为计划、执行、验证和输出四个阶段，并通过可观察的运行记录帮助你跟踪研究过程与来源。它提供了：
- 多代理协作（规划、执行、验证、回答）
- 可插拔的工具与数据源（网页检索、PDF、Notion、邮件等）
- 可追溯的研究过程与引用
- 适合“需要证据与可复核结论”的场景

源码与说明：
- GitHub: https://github.com/virattt/dexter
- 官方文档: https://dexter-phi.vercel.app

## 适用场景
- 行业/公司/产品调研与竞品分析
- 技术路线与论文综述
- 市场规模与趋势验证
- 需要多来源交叉验证的内容整理

## 快速上手
> 以官方 README 与文档为准。

1. 克隆仓库并安装依赖
   ```bash
   git clone https://github.com/virattt/dexter
   cd dexter
   pnpm install
   ```

2. 配置环境变量
   - 需要配置大模型、检索与工具相关的 API Key
   - 具体键名以 `.env.example` 与文档为准

3. 启动
   ```bash
   pnpm dev
   ```

4. 进入 Web UI 进行研究任务

## 工作原理（简版）
1. **规划（Planner）**：把问题拆成多个子问题与检索目标，确定信息来源与验证路径。
2. **执行（Executor）**：调用搜索、网页、PDF、数据库等工具拉取材料，完成初步信息收集。
3. **验证（Verifier）**：对关键结论进行交叉验证，发现冲突或缺口再回到执行补全。
4. **回答（Answerer）**：输出结构化结论、引用来源与可复核链路。

## 核心能力
- 多代理协同推进复杂任务
- 支持多种数据源与工具连接
- 研究过程可追踪、可回放
- 适合输出“带证据链”的内容

## 技术栈与形态
- 前端：Next.js
- 运行：Node.js / pnpm
- 后端/工具：Agent 工具集合（搜索、文档、第三方工具）

## 注意事项
- 结果质量取决于数据源与模型能力
- 仍需人工判断与复核关键结论
- 长链路任务建议拆分并设置验证点
