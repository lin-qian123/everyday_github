# codegraph

## 定位
`codegraph` 提供“预索引代码知识图谱”，目标是减少代理在代码探索时的工具调用与 token 消耗。

- 仓库：<https://github.com/colbymchenry/codegraph>
- 观察时间星标：约 2.5k stars

## 用法
- 安装后在目标仓库执行初始化（如 `codegraph init` 一类流程）。
- 让代理优先查询图谱而非全量文件扫描。
- 在大型仓库中对比接入前后的调用次数与探索时延。

## 原理
- 预先抽取符号关系、调用图与结构化索引。
- 将“在线遍历文件”转为“图查询 + 精准读取”，减少盲扫。
- 与代理框架耦合时，可显著压缩探索阶段的上下文消耗。

## 价值
- 对中大型代码库的首轮理解效率提升明显。
- 在 token 成本敏感场景有直接工程收益。

## 风险边界
- 索引时效性是关键，代码快速变更会导致图谱陈旧。
- 多语言/动态特性项目中，静态图谱可能存在盲区。

## 补充建议
- 将重建索引纳入 CI 或 pre-merge 检查策略。
- 对关键路径保留人工 spot-check，避免过度信任图谱推断。
