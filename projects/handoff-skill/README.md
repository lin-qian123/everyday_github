# handoff-skill

## 定位

`handoff-skill` 是把当前 Claude 对话整理为可接续交接文档的 skill，目标是让另一位人或任意 LLM 快速恢复任务上下文。

截至 2026-07-23，GitHub API 显示仓库创建于 2026-07-22，约 20 stars、7 forks；属早期开发者信号。

## 用法

按仓库安装说明把 skill 加入 Claude Code，在需要切换会话、模型或维护者时运行交接流程，并将产物随任务文件保存。

## 原理

它将目标、已做工作、证据、未决问题和下一步动作编码为结构化 handoff，而不是依赖聊天记录的隐式上下文。

## 价值

- 适合长任务、配额中断和多 agent 轮换。
- 将可复核事实与推测分开，降低接手者重复探索成本。

## 风险边界

- 交接文件可能包含凭据、私有路径或敏感业务上下文，提交前必须脱敏。
- 产物质量仍取决于执行者是否如实记录失败与未知项。

## 补充建议

把 handoff 视为 `AGENTS.md`、测试结果和 Git 状态的补充，不要把它当作唯一事实源。
