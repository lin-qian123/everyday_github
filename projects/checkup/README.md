# checkup

## 定位

`checkup` 是 Claude Code skill，用经验检查发现失效链接、陈旧徽章、依赖和 CI 衰退以及文档矛盾，并给出等级与优先修复项。

截至 2026-07-23，GitHub API 显示其创建于 2026-07-22，约 1 star；属早期开发者信号。

## 用法

在目标仓库运行检查流程，让它产生证据与修复建议；应先限制其自动修改范围。

## 原理

把仓库健康度拆成可观察证据：链接、依赖、CI、文档与代码一致性，而非仅生成笼统审计报告。

## 价值

- 对长期维护的 agent 项目有助于发现“能运行但已衰败”的外围问题。
- 检查结果可转成可排期的维护任务。

## 风险边界

- 自动诊断的结论仍需人工复核，尤其是依赖升级建议。
- 外部链接扫描可能产生网络噪声或误报。

## 补充建议

让报告附带命令输出与失败样本，并将修复拆分为小提交。
