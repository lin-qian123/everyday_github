# UI-TARS Desktop

项目简介
- UI-TARS Desktop 是字节跳动开源的桌面 GUI 智能体产品，面向“操作真实桌面应用”的场景，把大模型的多模态能力用在截图理解、界面定位与动作执行上。
- 目标是让 AI 像人一样使用电脑：看见屏幕 → 理解界面 → 规划动作 → 点击/输入/滚动 → 反馈纠错。

为什么值得关注
- “软件即界面”的传统应用不需要改造，直接让 AI 操作现有 GUI。
- 适合自动化办公、测试脚本、重复操作、跨应用串联流程等。

核心能力（基于官方说明整理）
- 真实桌面环境：不是浏览器沙盒，直接操作桌面应用。
- 视觉理解 + 动作执行：基于截图和 UI 元素定位进行点击/输入/拖拽。
- 任务分解与纠错：多步任务可自动拆分，失败时根据反馈重试。

如何使用（入门）
1. 下载客户端：在 GitHub Release 或官网获取对应系统安装包。
2. 配置模型：设置你可用的多模态大模型（VLM）提供方与密钥。
3. 打开任务：在 UI-TARS Desktop 中输入目标，例如“打开 Excel，创建一个新表并填入示例数据”。
4. 观察执行：AI 将自动操作鼠标/键盘，必要时会请求确认。

使用技巧
- 任务描述尽量具体，包含目标与约束（例如“不要关闭现有窗口”）。
- 对复杂任务，先给出整体目标，再补充步骤与注意事项。
- 允许 AI 先“探索界面”，再执行关键操作，成功率更高。

原理概述（简化版）
- 感知：周期性截屏 + OCR/视觉编码，形成“可读界面”。
- 规划：大模型基于截图与任务目标生成下一步动作序列。
- 执行：通过系统级事件注入完成点击/输入/滚动等操作。
- 反馈：再次截屏对比，判断是否达成，必要时修正计划。

适用场景
- 跨软件流程：例如“从邮件下载附件 → 打开 → 填表 → 上传”。
- 自动化测试：复现固定步骤并截图留证。
- 个人效率：批量处理、重复输入、日常办公。

注意事项
- 需要授予系统级辅助功能/屏幕录制权限（macOS/Windows）。
- 任务执行中尽量避免手动抢占鼠标键盘，以免打断。
