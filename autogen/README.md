# autogen

## 定位
`microsoft/autogen` 是多智能体应用框架，目标是把“单模型对话”升级为“多角色协作 + 工具编排 + 可控执行”的工程体系。

## 用法
1. 按官方文档安装依赖并配置模型后端。
2. 定义代理角色（例如规划、编码、审查、执行）及其消息协议。
3. 绑定工具调用（代码执行、检索、外部 API）并设置停止条件。
4. 用小任务做闭环验证后，再扩展到复杂流程或企业场景。

## 原理
- 通过多代理会话协议组织任务分工与协商。
- 使用工具调用和函数接口把“推理”连接到“可执行动作”。
- 通过会话管理与终止条件控制成本、质量和安全边界。

## 价值
- 适合需要流程化自动化的团队（研发、数据、运营协同）。
- 对“复杂任务拆解 + 并行推进 + 审核收敛”场景更友好。
- 生态成熟、社区案例多，便于快速试错与复用组件。

## 风险边界
- 多代理系统调试复杂度高于单代理，容易出现循环对话与成本失控。
- 若停止规则与权限边界设计不当，会造成错误放大。
- 不同模型后端行为差异明显，迁移时需重新验证。

## 补充建议
- 先做“最少代理数”的架构验证，再逐步增加角色复杂度。
- 对每个代理定义清晰 KPI（响应质量、成本、失败率）。
- 建立可观测性：日志分层、轨迹回放、失败案例库。
