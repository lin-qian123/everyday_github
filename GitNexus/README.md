# GitNexus

一个面向 LLM / Agent 的代码理解与检索工具：把代码库分析成可查询的“知识图谱”，并在浏览器内完成解析、索引与可视化，让模型更容易定位 API、调用链、依赖关系与关键语义。核心目标是“让 LLM 读懂仓库结构，而不是只做全文检索”。

## 它解决什么问题
- 传统全文检索对复杂工程（多语言、多模块）理解有限，Agent 往往找不到真正的入口与依赖关系。
- GitNexus 用语法树 + 代码图谱结构化代码，给 LLM 一个更可推理的上下文视图。

## 核心原理（简要）
- **结构化解析**：利用 Tree-sitter WASM 解析代码为 AST，并抽取符号/引用。
- **知识图谱建模**：把类、函数、模块、调用关系与依赖关系写入浏览器内嵌图数据库（KùzuDB WASM）。
- **本地优先**：分析与索引都在浏览器侧完成，减少代码与密钥外泄风险。

## 如何使用

### 方式一：浏览器内分析（推荐）
1. 运行项目（本地开发模式）。
2. 在页面中输入 GitHub 仓库地址或上传 ZIP。
3. 选择过滤条件后点击 Analyze，等待图谱生成。
4. 在图谱与聊天界面中检索实体、调用链与依赖关系。

### 方式二：本地开发运行
```bash
git clone https://github.com/abhigyanpatwari/GitNexus.git
cd GitNexus
npm install
npm run dev
```

## 典型场景
- 复杂项目的快速 onboarding：找到最关键的入口与依赖。
- 生成结构化上下文：把调用链、实体关系作为提示输入模型。
- 图谱检索优先：避免只靠 embedding 检索的“语义偏差”。

## 注意事项
- 初次索引大型仓库会有一定耗时；可先从小仓库或子目录试运行。
- 注意浏览器内存占用，必要时限制文件数量或目录范围。
