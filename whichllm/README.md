# whichllm

## 定位
`whichllm` 用于根据本机硬件条件推荐“可运行且效果更优”的本地模型，而不是只按参数规模挑模型。

- 仓库：<https://github.com/Andyyyy64/whichllm>
- 观察时间星标：约 840 stars

## 用法
- 命令行运行 `whichllm` 获取推荐列表。
- 可用 `--gpu` 模拟指定显卡（如购机前规划）。
- 结合 `run`、`--json` 等模式接入自动化脚本或实验流程。

## 原理
- 结合硬件检测（GPU/CPU/RAM）与模型元数据筛选候选。
- 引入多来源 benchmark 与置信度折扣，不仅看“能装下”，还看“实测表现”。
- 使用实时数据源与缓存机制平衡新鲜度与速度。

## 价值
- 减少本地模型试错成本，尤其对资源受限设备更实用。
- 适合作为私有化部署前的模型选型入口。

## 风险边界
- 排名质量受外部 benchmark 完整性与时效影响。
- 实际推理体验仍会受驱动、后端实现与量化方案波动影响。

## 补充建议
- 保留“任务级 A/B 测试”而非只看总分。
- 对关键任务（代码、数学、视觉）分场景建立自己的小评测集。
