# oh-my-pi

## 定位
`can1357/oh-my-pi` 是终端优先（terminal-first）的 AI Coding Agent，聚焦真实开发过程中的多工具协作、插件扩展与多模型/多搜索后端接入。

## 用法
1. 按官方文档安装并初始化运行环境。
2. 配置你使用的模型或搜索提供方（支持 API Key 或部分 OAuth 路径）。
3. 在真实项目中执行编辑、搜索、调用外部工具、子代理分工，并结合日志检查稳定性。

## 原理
- 以 CLI/TUI 交互为主，强调开发循环中的低上下文切换成本。
- 支持插件化扩展与多 provider 路由，把搜索、代码编辑、文档检索等能力统一到一个代理工作台。
- 通过结构化处理外部内容（如代码托管站点、包管理站点、技术社区）提升可引用性与可追溯性。

## 价值
- 对重度终端用户和自动化开发流程友好。
- 在多模型并行评估、工具链组合实验方面可操作性较强。
- 适合构建“本地优先 + 可扩展”编码代理工作流。

## 风险边界
- 多 provider 与插件叠加后，配置复杂度和排障成本会上升。
- 第三方后端的可用性与限额会直接影响体验。
- 在企业环境中需关注密钥管理、审计与数据边界。

## 补充建议
- 先用最小配置跑通核心链路，再逐步增加 provider 和扩展能力。
- 给常用任务设计固定脚本/模板，减少人为操作波动。
- 在团队使用前，先建立“安全与合规基线”：日志留存、权限最小化、敏感信息脱敏。
