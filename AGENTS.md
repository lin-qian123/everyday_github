# AGENTS 指令（everyday_github）

## 目标

本仓库用于每日跟踪 AI 热点与 GitHub 高热项目，并沉淀中文项目解读与热点日报。

## 日常执行规则

1. 每次执行前检查并维护：`AGENTS.md`、`README.md`、`TODO.md`。
2. 直接在当前分支更新，不创建新分支。
3. 每日新增或更新：
   - `daily/YYYY-MM-DD/ai-hotspots.md`
   - 当日新上榜 AI 项目的 `projects/<project>/README.md`（目录名与项目名保持一致）。
4. 项目说明必须包含：定位、用法、原理、价值、风险边界、补充建议、参考资料。
5. 热点日报至少覆盖：GitHub、X、Instagram、YouTube，并给出可追溯链接。
6. 所有项目仓库目录统一放在 `projects/` 下；后续新增、更新、去重检查都以 `projects/` 为准，不再直接放在仓库根目录。
7. 每日处理新项目时，必须为项目选择合适分类，并同步维护首页 `README.md` 的分类导航；优先复用现有分类，仅在明显必要时新增分类。
8. 每次自动化成功完成后，默认执行提交并推送到 `origin/master`；若无实际文件变化，则跳过提交并在结果中明确说明。

## 首页分类基线

首页 `README.md` 当前优先使用以下分类：

- `Coding Agents 与终端助手`
- `Agent 框架与技能生态`
- `记忆层与个人 AI 基础设施`
- `RAG、检索与知识处理`
- `前端、UI 与 Agent 交互层`
- `语音、视频与多模态`
- `模型、训练与推理基础设施`
- `办公、商业与行业应用`

新增项目时应先判断是否可放入上述分类，并同步更新首页中的“项目 / 简介 / 链接”索引表。

## 质量标准

- 结论必须有来源，不凭空判断。
- 用中文写作，优先工程实践价值。
- 对争议性话题明确不确定性与证据等级。
- README 不是静态说明，而是仓库首页导航；若新增项目影响分类代表性，必须同步更新首页。
