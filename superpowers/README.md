## 这是什么

`superpowers` 是一个 Agent 开发方法论与技能框架仓库，强调在编码前先抽取需求、生成可审阅规格、再进入实现计划与执行，从流程上约束 AI 代理“先想清楚再动手”。

## 热度信号

1. GitHub 仓库页抓取快照显示：约 `153k stars`、`13.3k forks`，讨论与协作活跃。
2. OSSInsight 趋势榜近期也将其列为高关注项目之一，说明它不仅“历史热”，也有持续新增关注。

## 怎么用（最短路径）

### 1) 克隆仓库

```bash
git clone https://github.com/obra/superpowers.git
cd superpowers
```

### 2) 先用现成技能跑一条端到端任务

从仓库中的技能目录挑一个与你当前问题最接近的工作流，按其步骤先做一次完整任务（需求澄清 -> 方案 -> 实施 -> 验证）。

### 3) 纳入你的日常代理工具链

将可复用技能迁移到你的全局技能目录，并在团队约定中固定触发条件；升级时可使用仓库提供的更新方式（例如 `/plugin update superpowers`）。

## 核心原理

1. 把“软件工程流程”拆成可组合技能。
2. 在任务入口强制澄清目标与验收标准，减少盲写代码。
3. 让代理按阶段推进，并在阶段间保留可审计产物。
4. 用技能复用替代一次性提示词，持续累积团队能力。

## 适用场景

1. 中大型项目的长期迭代。
2. 需要多人/多代理协同的复杂任务。
3. 希望建立稳定交付质量基线的团队。

## 价值与风险

价值：
- 通过流程化减少“看似快、实则返工”的问题。
- 技能沉淀能随项目成长而复利。

风险：
- 初期上手成本高于直接提示词驱动。
- 若流程过重，小任务可能被不必要地放慢。

## 我认为有价值的补充材料

1. 先在高风险模块试点，再逐步扩散到全仓库。
2. 给技能加“轻量/标准/严格”三档执行深度。
3. 结合仓库自动化任务定期回收低价值流程节点。
