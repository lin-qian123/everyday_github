# pi-mono

概览
pi-mono 是一个多模态 RAG 项目，强调“带 REPL 的多模态检索增强生成”。仓库介绍其目标是把图像与文本统一进入检索流程，并基于多模态模型完成推理与回答。

核心能力
- 多模态输入：支持图像与文本进入检索流程。
- RAG + REPL：把检索结果交给可交互的执行环境（REPL）进行更复杂的推理与验证。
- 默认基模建议：项目说明中使用 `phi-4-multimodal-instruct` 作为示例模型。

原理与架构（基于公开说明的归纳）
pi-mono 将图像与文本统一编码为可检索表示，先做多模态检索得到相关上下文，再交给多模态模型进行生成。REPL 允许在生成过程中执行结构化推理或计算验证，以提高答案可靠性。

快速上手
- 按仓库说明配置环境与依赖。
- 准备图像或文本素材作为检索库。
- 运行项目提供的示例脚本，启动 REPL 体验多模态问答。

使用场景
- 多模态文档库（图片 + 说明文）检索问答。
- 需要可解释/可验证推理过程的多模态应用。

风险与注意事项
- 多模态模型对显存与算力要求较高，注意本地环境配置。
