# memU

- 项目地址：https://github.com/NevaMind-AI/memU
- GitHub 热度（2026-03-14 抓取快照）：约 `4.5k stars`、`307 forks`
- 定位：给 LLM/AI Agent 提供“可追溯、可演化”的长期记忆基础设施（支持多模态输入、双检索策略和分层记忆组织）。

## 这是什么
`memU` 是一个面向 Agent 后端的记忆系统，不是单纯“向量库包装”。

它把对话、文档、图片、音视频等输入统一处理成三层结构：
- Resource：原始多模态数据
- Item：提取后的离散记忆单元（偏好、习惯、观点、技能等）
- Category：归纳后的主题记忆（如 `preferences.md`、`work_life.md`）

核心价值是“记忆可追溯 + 可总结 + 可持续更新”，方便 Agent 在长期会话里保持一致性。

## 怎么用（最短路径）

## 1) 安装
```bash
pip install memu
```

## 2) 启动服务
```bash
memu
```

## 3) 配置模型与存储
在项目根目录准备 `.env`（按官方文档填写 OpenAI/Anthropic/Qwen 等模型配置，及数据库/存储配置）。

## 4) 接入你的 Agent
常见接入方式：
1. 在对话结束后把新消息写入 memU（作为 Resource）。
2. 在回复前从 memU 检索相关 Item/Category。
3. 将检索结果注入系统提示词或工具上下文。
4. 定期执行“记忆压缩/整理”，减少噪声。

## 原理是什么（工程视角）

## 1) 分层记忆文件系统
使用 `Resource -> Item -> Category` 的层次化结构，把“原始事实”与“抽象认知”分开存储。
这样做的好处是：
- 能回溯来源，降低幻觉记忆风险
- 能在高层做摘要，降低上下文成本

## 2) 双检索策略
- Embedding/RAG 检索：快，适合高频在线召回
- 非 embedding 的 LLM 检索：慢但语义更深，适合复杂查询

项目强调两种策略并行，以平衡速度与理解深度。

## 3) 多模态统一抽取
对文本、图像、视频、音频做统一抽取后写入同一记忆层级，支持跨模态查询。
例如：用户发来截图 + 语音说明，最终都能汇总到同一类偏好记忆。

## 4) 自演化机制
记忆结构会根据新输入和检索使用情况持续调整，让“高价值记忆”更容易被召回。

## 适用场景
- 长周期 Copilot/Agent（需要记住用户风格、约束、历史决策）
- 私人知识助理（长期积累学习轨迹）
- 客服/销售 Agent（跨会话维持客户画像）
- 多模态应用（文本 + 图片 + 音频 + 视频混合输入）

## 风险与边界
- 记忆系统会放大隐私与合规问题：必须做 PII 过滤、访问控制和删除机制。
- 长期记忆易“过拟合”用户早期偏好：建议加时间衰减与人工校正。
- 多模态抽取质量依赖底层模型，需做离线评估和回归测试。

## 今天建议
1. 先做一个最小闭环：写入 20 条历史对话 -> 检索 -> 观察回复一致性变化。
2. 为每条记忆保存 `source_id` 与时间戳，保证可审计。
3. 给生产环境加“禁写开关”和“遗忘策略”，避免脏数据累积。
