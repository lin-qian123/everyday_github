# local-deep-research

概览
Local Deep Research（LDR）是一个强调“本地优先、数据可控、可追溯引用”的 AI 研究助理框架。它把多来源检索、证据整合与报告生成做成一条可配置流水线，适合研究/分析类深度任务。

核心价值
- 支持本地与云模型混合，部署灵活。
- 可接入多种检索源（网页、学术库、私有文档）。
- 输出带引用，利于复核与二次写作。
- 以隐私与本地数据所有权为主要卖点。

如何使用
1. 按仓库文档启动依赖服务（如检索引擎、模型服务）。
2. 配置 LLM、检索源与数据目录。
3. 先跑快速总结，再切换到深度研究模式。
4. 将高价值结果沉淀进本地知识库，形成持续复用。

原理拆解
- 检索规划：先拆分问题，再选择合适数据源。
- 证据聚合：多轮抓取后进行来源对齐与冲突处理。
- 报告合成：按结构化模板生成带引用结论。
- 本地存储：文档索引与历史结果沉淀为可检索资产。

适用场景
- 学术综述、竞品调研、技术路线评估。
- 对数据外发敏感的企业内部研究。
- 需要“可复核链路”的知识生产团队。

补充观察
- 深度研究赛道的关键不是“搜索次数”，而是“检索策略质量 + 证据一致性 + 结论可解释性”。
- LDR 适合作为组织内部“研究操作系统”的底座，而非一次性问答工具。
