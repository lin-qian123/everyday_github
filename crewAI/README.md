# crewAI

## 定位
`crewAIInc/crewAI` 是多智能体协作框架，目标是把“一个大模型”升级为“角色分工 + 工作流编排”的可执行系统。

## 用法
1. 按文档安装 `crewai` 及依赖。
2. 定义角色（如研究员、实现者、审校者）和工具权限。
3. 配置任务流（顺序、并行、条件分支）并执行。
4. 将日志与评估接入可观测体系，持续调 prompt/工具路由。

## 原理
- 将任务拆分为多个 Agent 角色，每个角色绑定能力边界。
- 用编排层控制任务传递、状态流转和重试机制。
- 借助工具调用实现检索、代码执行、外部系统交互。

## 价值
- 提升复杂任务的可分工性与可追踪性。
- 相比单 Agent，流程更容易做质量治理。
- 适合报告生成、研发协作、运营自动化等场景。

## 风险边界
- 角色过多会显著提高系统复杂度和调试成本。
- 多轮协作可能放大早期错误并导致偏航。
- 工具权限若隔离不严，存在越权调用风险。

## 补充建议
- 从 2-3 个角色起步，先验证 ROI 再扩展。
- 强制加入评估节点（事实核验、格式校验、失败回退）。
- 用结构化日志记录每步输入/输出，便于复盘。
