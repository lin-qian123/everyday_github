# spec-kit（中文解读）

- GitHub：<https://github.com/github/spec-kit>
- 标签：规范驱动开发、AI 协作、工程流程标准化

## 1. 项目定位

`spec-kit` 是 GitHub 推出的规范化开发工具包，核心目标是把“想法/需求”先落到可审阅的规范，再进入实现阶段，减少 AI 生成代码时的方向偏差与返工。

## 2. 用法（建议流程）

1. 按仓库文档创建或选择一个项目工作目录。
2. 使用 `spec-kit` 提供的模板/命令产出需求说明、技术约束与验收标准。
3. 让 AI 助手基于这份规范生成实现计划与代码，而不是直接从一句 prompt 开始写。
4. 在提交前用规范对照检查：功能是否覆盖、边界是否违反、风险是否已声明。

## 3. 原理拆解

- 核心机制：把“自然语言需求”转为“结构化约束文档”。
- AI 协作逻辑：模型不再只依赖上下文猜测，而是围绕明确字段（目标、非目标、接口、风险）工作。
- 结果：提高跨人/跨模型的一致性，降低“看起来能跑但不满足真实需求”的概率。

## 4. 工程价值

- 对团队：统一评审语言，降低沟通损耗。
- 对个人开发者：在快速迭代中保留基本可追溯性。
- 对 AI 编程：显著提升可控性，尤其适合中等复杂度项目。

## 5. 风险边界

- 规范写得差，后续 AI 产出也会系统性偏离。
- 小型一次性脚本若强行引入全流程，可能增加流程负担。
- 该工具不替代架构判断与生产环境验证。

## 6. 补充建议

- 为规范增加“不可触碰约束”字段（安全、合规、成本上限）。
- 将规范与 CI 校验结合，避免“文档和代码长期漂移”。
- 在仓库中维护变更日志，记录需求为何变化。
