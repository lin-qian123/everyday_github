# rtk

## 定位
`rtk-ai/rtk` 是一个面向开发命令场景的 CLI 代理/代理层，主打降低 LLM token 消耗。

## 用法
1. 按官方指南安装单二进制程序。
2. 在本地常用开发命令前接入代理层。
3. 对比接入前后的 token 消耗、延迟与成功率。

## 原理
- 对命令交互进行压缩与路由优化。
- 通过缓存、裁剪和上下文代理降低重复 token。
- 保持 CLI 习惯不变，尽量实现“低侵入接入”。

## 价值
- 在高频 CLI 场景下可显著降低成本。
- 对长会话开发任务更友好。
- 对团队预算管理有直接帮助。

## 风险边界
- 极端压缩策略可能损失上下文细节。
- 代理层故障会影响主流程可用性。
- 成本下降需要真实负载验证，不能只看宣传值。

## 补充建议
- 先在只读或低风险命令上灰度。
- 记录“节省比例 vs 失败率”双指标。
- 为关键任务配置直连回退开关。
