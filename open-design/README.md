## 1. 项目定位
`open-design` 是一个面向设计产物生成的开源 AI 工作台，主打“本地优先 + BYOK + 多代理 CLI 接入”。
它的核心不是单一模型能力，而是把设计任务拆成可复用的技能与设计系统，直接生成原型、网页、幻灯片等交付物。

## 2. 基础用法
1. 克隆仓库并安装依赖（README 推荐 `pnpm` 工具链）。
2. 启动本地开发循环（项目文档中给出 `pnpm tools-dev` 入口）。
3. 在提示词里描述目标产物，例如：
   - “做一份种子轮路演 deck，偏杂志风，输出可导出 PPTX 的版本”。
4. 选择视觉方向与设计系统，审阅自动生成的 artifact，再进行多轮迭代。
5. 按需导出 HTML/PDF/PPTX 等格式，并进入人工审校流程。

## 3. 原理理解
- 代理编排层：自动识别本地可用的 coding/design agent CLI，把它们作为执行引擎。
- 技能层：把“提示词工程 + 检查清单 + 产物约束”封装成技能模板，降低随机性。
- 设计系统层：沉淀品牌色、字体、版式规则，统一生成物风格。
- 产物层：将输出约束为可渲染 artifact，而非仅文本描述，便于直接评审和导出。

一句话：它在做“把 AI 设计从聊天升级为工程化交付流水线”。

## 4. 价值与适用场景
- 适合需要快速产出视觉方案的产品/运营/增长团队。
- 适合“先出可演示稿，再交专业设计师精修”的协作模式。
- 对于多品牌、多业务线团队，设计系统复用价值明显。

## 5. 风险与注意点
- 设计自动化会带来审美同质化风险，需要人类创意把关。
- 产物可用性不等于可上线质量，仍需可访问性、版权与品牌一致性检查。
- 多代理执行链路中，权限边界和敏感数据脱敏必须前置。
