# hermes-webui

## 项目定位
`hermes-webui` 是 Hermes Agent 的 Web 交互层，目标是把终端 Agent 能力搬到浏览器和移动端，便于团队协作与远程访问。

- GitHub：https://github.com/nesquena/hermes-webui
- 适用人群：希望把 AI Agent 接入内部工具台、给非 CLI 用户开放入口的团队。

## 用法

1. 阅读官方仓库 README 与部署说明（Docker/本地模式）。
2. 配置 Agent 后端连接、鉴权信息与会话存储。
3. 通过 Web 界面发起任务、查看日志和结果。

典型落地方式：
- 内网部署：用于研发/运营团队共享 Agent。
- 演示环境：用于产品验证和业务方试用。

## 原理

- 前端层：提供会话界面、任务状态与结果展示。
- 编排层：把用户输入转换为 Agent 可执行任务。
- 后端层：连接模型与工具调用链，执行多步任务。
- 状态层：保存会话历史、上下文和执行记录。

## 价值

- 降低 Agent 使用门槛，不再局限终端用户。
- 便于团队共享 Prompt、工作流与执行结果。
- 可作为“Agent 产品化”的第一层交互壳。

## 风险边界

- 远程执行能力必须配套最小权限策略。
- 会话与日志中可能包含敏感信息，需脱敏与审计。
- 多用户并发下，需要资源隔离和配额控制。

## 补充建议

- 上线前补齐三类机制：SSO/权限、审计日志、速率限制。
- 对高风险工具调用设置二次确认（如写文件、外部调用）。
- 建议增加“只读模式”和“沙箱模式”两档运行策略。
