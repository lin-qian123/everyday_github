# aider

## 定位
`paul-gauthier/aider` 是面向开发者的终端 AI 编码助手，强调在本地 Git 仓库中进行可追踪、可提交的代码修改。

## 用法
1. 安装 `aider` 并配置模型 API Key。
2. 在目标仓库内启动，指定要编辑的文件。
3. 通过对话驱动改动，并结合测试命令迭代。
4. 使用 Git 提交把改动纳入常规代码审查流程。

## 原理
- 基于仓库上下文将相关文件提供给模型生成补丁。
- 与 Git 工作流结合，保留变更可追溯性。
- 通过“提问-改动-测试-再改动”形成闭环。

## 价值
- 提升小步迭代效率，尤其适合重构和脚手架任务。
- 与现有工程工具链兼容，接入成本低。
- 便于把 AI 产出纳入代码审查与质量门禁。

## 风险边界
- 模型可能误解上下文，改出表面可运行但语义错误的代码。
- 对大仓库或跨模块改动仍需人工架构把关。
- 外部 API 依赖带来成本、速率和合规约束。

## 补充建议
- 强制“先测后提”的最小门禁策略。
- 对高风险目录启用人工审批与双人复核。
- 记录常见失败模式，沉淀团队提示词模板。
