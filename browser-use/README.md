# Browser-Use

## 简介
Browser-Use 是一个让 AI 代理可控、可靠地使用真实浏览器完成任务的开源工具。它把“浏览器状态 + 可操作动作”暴露给 LLM 代理，让代理能像人一样点击、输入、滚动、下载并完成多步骤网页流程。

适合场景：
- 资料检索与对比（跨站点搜索、比价、对比功能）
- 表单填写或重复性流程自动化
- 面向真实网页的端到端任务（下单流程验证、运营分析等）

## 核心原理（高层）
- 代理（Agent）负责理解目标、规划步骤，并调用浏览器动作工具。
- 浏览器（Browser）持续提供页面状态（DOM、截图、可交互元素等），并执行点击/输入/滚动等动作。
- LLM 根据最新页面状态进行下一步决策，形成“观察 -> 推理 -> 行动”的闭环。

## 快速上手
以下是典型本地使用路径（不同环境略有差异）：

1. 安装依赖
```
uv pip install browser-use
```

2. 准备浏览器（如 Playwright/Chromium）
```
playwright install chromium
```

3. 运行一个简单 Agent
```python
from browser_use import Agent

agent = Agent(
    task="在某个网站搜索‘AI 代理’，并打开结果第一条",
    llm="openai:gpt-4.1"
)

agent.run()
```

## 常见使用方式
- CLI 执行任务：把目标描述直接交给 Agent
- 脚本化：把任务拆成多段，或自定义工具函数
- 多步骤流程：登录、搜索、过滤、导出等

## 关键能力
- 真实浏览器执行：适配复杂网页和动态内容
- 低延迟交互：代理在本地或靠近浏览器运行
- 可扩展工具：可以加入自定义动作或验证步骤

## 评价与注意点
- 适合“网页即系统”的场景，但仍受限于网页变化与登录验证
- 复杂任务建议加入“中间检查点”或明确的成功条件
- 对稳定性要求高时，建议加断言和重试策略
