# semble

## 定位

`MinishLab/semble` 是一个面向 AI 编码代理（如 Claude Code、Codex、Cursor）的高性能代码检索工具，目标是在“大仓库 + 多轮代理调用”场景中降低检索 token 与工具调用成本。

## 用法

1. 从仓库 README 获取安装方式（通常包含本地服务或 CLI 模式）。
2. 对目标代码库做索引或接入其预索引能力。
3. 在代理工作流中把“全仓搜索”替换为 `semble` 查询：
   - 先语义检索定位候选文件
   - 再精读少量片段
   - 最后执行改动与验证

## 原理

从公开说明可见，`semble` 走的是“嵌入检索 + 代码结构感知”的路线：

- 预处理：把仓库切分为可检索代码单元并建立索引。
- 查询阶段：将自然语言任务转换为语义查询，先召回高相关片段。
- 压缩上下文：把候选片段精简后返回给代理，减少盲目 `grep + 全文读取`。

## 价值

- 对 AI 编码代理：显著减少无效读取与上下文膨胀。
- 对工程团队：在大仓库里提升“定位-修改-验证”闭环速度。
- 对成本控制：在高频自动化任务中降低 token 和工具调用成本。

## 风险边界

- 检索召回率并非 100%，仍需人工复核关键改动。
- 索引新鲜度会影响结果质量，仓库频繁变更时需重建或增量更新。
- 热度上升不等于稳定可用，需关注 issue 响应、版本节奏与基准测试口径。

## 补充建议

- 在 CI 中加入“关键路径双检索”（语义检索 + 关键词检索）防漏召回。
- 对核心模块建立固定回归查询集，监控检索质量漂移。
- 与代理提示词配套：明确先检索再改动，避免上下文超载。
