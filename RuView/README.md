# RuView

## 项目定位

- GitHub：<https://github.com/ruvnet/RuView>
- 定位：通过 WiFi CSI（Channel State Information）实现无摄像头的空间感知与体征监测。
- 当前信号（2026-05-16 抓取）：约 57.5k stars。

## 用法

### 体验路径（建议）

1. 先用 Docker/模拟数据跑通流程。
2. 再接入 ESP32-S3 节点进行实时采集。
3. 最后按场景启用检测模块（占用、呼吸/心率、活动识别）。

### 硬件关键点

- 高级能力依赖 CSI 采样硬件（如 ESP32-S3）。
- 单节点能力有限，推荐多节点或结合额外设备。

## 原理

- **信号采集层**：通过 WiFi 通道子载波变化捕捉人体扰动。
- **特征与融合层**：多链路/多通道时序融合，提取动作和生理节律信息。
- **任务层**：输出 presence、vital signs、activity 等高层语义事件。

## 价值

- 适用于对隐私敏感的场景（不部署视频摄像头）。
- 在黑暗、遮挡条件下仍可维持一定监测能力。

## 风险边界

- 体征与姿态相关能力对硬件部署和环境噪声高度敏感。
- 医疗级结论不可直接外推，需严格临床/法规路径。
- 大规模部署需要关注误报成本与维护复杂度。

## 补充建议

- 先定义可接受误报率，再设计阈值与告警策略。
- 对不同房型做场景基线标定，降低迁移失真。
- 涉及安防/健康决策时，必须保留人工复核环节。
