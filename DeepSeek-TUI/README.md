# DeepSeek-TUI

概览
DeepSeek-TUI 是一个面向开发者的终端 AI 编码代理，核心定位是把“对话 + 改代码 + 执行命令 + 审批闸门”放进同一个 TUI 工作流。它强调本地项目内的高频迭代，适合希望在命令行中完成完整编码闭环的用户。

项目价值
- 终端内完成需求到补丁的闭环，减少 IDE 与网页来回切换。
- 支持模型与思考强度动态选择，兼顾速度与质量。
- 对文件编辑与命令执行保持可见、可审计，降低误操作风险。

如何使用
1. 安装后在终端运行 `deepseek` 进入交互模式。
2. 在仓库目录中提出任务（如“修复测试失败并解释原因”）。
3. 审核工具调用和补丁内容，再决定是否应用。
4. 在自动模式下可让其按任务复杂度切换模型与推理等级。

原理拆解
- 调度层：根据任务复杂度和上下文长度，路由到不同模型配置。
- 执行层：通过受控工具链进行读写文件、运行 shell、检索信息。
- 安全层：以审批闸门和补丁可视化为核心，避免“黑箱式改动”。
- 反馈层：流式展示中间推理与执行结果，提升调试效率。

适用场景
- CLI-first 开发者的日常编码与重构。
- 需要在远程机/容器内快速完成修复。
- 希望保留可审计操作轨迹的团队协作。

补充观察
- 这类终端代理的竞争点正在从“能不能写代码”转向“工作流质量、审批体验、可回滚能力”。
- 在多代理并行和长上下文任务中，项目的稳定性与成本控制是长期壁垒。
