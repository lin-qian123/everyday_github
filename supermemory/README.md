# supermemory

## 项目定位
`supermemory` 是面向 AI 应用与 Agent 的记忆引擎项目，目标是提供高性能、可扩展的跨会话记忆基础设施。

- GitHub：https://github.com/supermemoryai/supermemory
- 适用人群：多 Agent 系统、需要长期上下文的应用团队。

## 用法

1. 按官方说明部署服务（本地或云端）。
2. 通过 API 写入会话片段、事实与检索标签。
3. 在 Agent 执行前按用户/任务上下文检索相关记忆。

建议优先定义记忆 schema（会话、实体、时间、权限）。

## 原理

- 写入链路：原始对话 -> 清洗摘要 -> 结构化存储。
- 检索链路：查询 -> 语义召回/过滤 -> 重排。
- 生命周期：记忆分层（短期/长期）、过期与删除策略。

## 价值

- 显著改善“多轮后遗忘”和上下文漂移问题。
- 支持跨工具、跨会话的用户偏好复用。
- 为 Agent 系统提供可观测、可治理的状态层。

## 风险边界

- 隐私数据沉淀后，删除一致性和合规压力增大。
- 错误记忆会被重复放大，造成长期偏差。
- 高召回率与低噪声难兼得，需要持续调参与评估。

## 补充建议

- 引入“记忆可信度分级”与人工纠错接口。
- 对敏感字段默认加密，并设置细粒度访问控制。
- 建立记忆回放审计，定期清理低价值陈旧记忆。
